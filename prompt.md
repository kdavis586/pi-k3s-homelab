# Background
## Project: Kubernetes on Pi
I want to set up Kubernetes on raspberry pis. The ultimate goal is to run lightweight homelab and potentially a github runner.

## Hardware
- 3x Raspberry Pi 4 Model B (2x 4GB RAM, 1x 8GB RAM) — treat all as 4GB for resource decisions
- 3x PoE hats installed on all Pis
- 1x TP-Link TL-SG605P (5-port unmanaged switch; ports 1-4 are PoE+, port 5 is standard data)
- 1x ATT router/modem combo (fiber, gateway: 192.168.1.254)
- 1x 1TB external hard drive (USB-C, will attach to the-bakery for Jellyfin media)
- 3x microSD cards (2x 64GB -> agents, 1x 32GB -> server)

## Network
- Home subnet: 192.168.1.0/24
- Gateway/DNS: 192.168.1.254
- Mac on WiFi: REDACTED

## Goals
- **Immediate**: Reproducible, git-driven cluster setup. Deploy changes via Ansible (no SD card shuffling after initial flash).
- **K8s distro**: K3s (chosen for Pi/ARM64 optimization, low memory footprint, built-in Traefik + local-path-provisioner)
- **Post-success**: Jellyfin on Kubernetes with media on the external USB drive, pinned to the-bakery

## Implementation Decisions Made
- **OS**: Ubuntu Server 24.04 LTS (64-bit) — flashed via Raspberry Pi Imager with NO OS customization (our cloud-init files handle everything)
- **cloud-init**: Files hand-copied to SD card `system-boot` partition before first boot. Two files per node: `user-data` and `network-config`
- **SSH keys**: Imported at boot via `ssh_import_id: [gh:kdavis586]` — no keys stored in repo
- **Static IPs**: Configured via cloud-init netplan (version 2 format)
- **Single source of truth**: `ansible/group_vars/all.yaml` — all hostnames/IPs live here. Run `make generate` after any change to propagate to cloud-init files, inventory, k8s manifests, and .make-vars
- **Config generation**: Jinja2 templates in `cloud-init/templates/` and `k8s/templates/`, rendered by `ansible/playbooks/generate-configs.yaml`
- **Repo**: https://github.com/kdavis586/pi-k8s (private)

## Repo Structure
```
ansible/
  group_vars/all.yaml      <- SINGLE SOURCE OF TRUTH (edit this, then run make generate)
  inventory.yaml           <- generated by make generate
  inventory.j2             <- template for inventory
  playbooks/
    generate-configs.yaml  <- renders all templates
    base-setup.yaml        <- run once after first boot (make setup)
    k3s-install.yaml       <- installs K3s server + agents (make install-k3s)
cloud-init/
  templates/
    user-data.j2           <- template for all node user-data
    network-config.j2      <- template for all node network configs
  user-data-the-bakery.yaml     <- generated
  network-config-the-bakery.yaml
  user-data-apple-pi.yaml       <- generated
  network-config-apple-pi.yaml
  user-data-pumpkin-pi.yaml     <- generated
  network-config-pumpkin-pi.yaml
k8s/
  templates/
    jellyfin-deployment.j2
    local-path-config.j2
  jellyfin/
    deployment.yaml        <- generated
  storage/
    local-path-config.yaml <- generated
make-vars.j2               <- template for .make-vars (gitignored, used by Makefile)
Makefile                   <- make generate | setup | install-k3s | deploy | ssh-<node>
```
