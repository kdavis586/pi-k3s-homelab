  <action>
Run `make bootstrap-flux` to install Flux on the cluster. This command:

1. Installs Flux v2.4.0 controllers into the flux-system namespace
2. Generates and commits gotk-components.yaml, gotk-sync.yaml, and kustomization.yaml to flux/flux-system/
3. Registers an SSH deploy key on GitHub for this repo (read-only)

**Prerequisites the executor must verify before running:**
- `GITHUB_TOKEN` env var is set with a fine-grained PAT (Administration R/W, Contents R/W, Metadata R)
- `flux` CLI is installed (`brew install fluxcd/tap/flux`)
- Cluster is reachable: `kubectl --kubeconfig ~/.kube/config-pi-k3s get nodes` returns nodes

**The command:**
```bash
make bootstrap-flux
```

The bootstrap command is idempotent — safe to re-run if it fails partway through.

**After bootstrap completes**, pull the latest commits because bootstrap pushes directly to main:
```bash
git pull origin main
```

Then verify the generated files exist:
```bash
ls flux/flux-system/gotk-components.yaml flux/flux-system/gotk-sync.yaml flux/flux-system/kustomization.yaml
```

**Important:** Bootstrap commits and pushes to the repo automatically. The executor does NOT need to commit these files — bootstrap does it.
