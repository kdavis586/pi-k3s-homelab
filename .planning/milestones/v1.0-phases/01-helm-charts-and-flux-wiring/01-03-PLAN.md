Create two distinct kustomization files (these serve DIFFERENT purposes — see research Pitfall 1):

1. Create `flux/apps/kustomization.yaml` — this is a PLAIN Kustomize config (NOT a Flux CRD) that tells kustomize-controller which files to process in this directory:
```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - jellyfin.yaml
  - pihole.yaml
```

2. Create `flux/flux-system/apps-kustomization.yaml` — this is a FLUX Kustomization CRD that tells Flux to reconcile the flux/apps/ directory, with a dependency on flux-system being healthy first:
```yaml
apiVersion: kustomize.toolkit.fluxcd.io/v1
kind: Kustomization
metadata:
  name: apps
  namespace: flux-system
spec:
  interval: 10m
  sourceRef:
    kind: GitRepository
    name: flux-system
    namespace: flux-system
  path: ./flux/apps
  prune: false
  dependsOn:
    - name: flux-system
```

IMPORTANT distinctions:
- `flux/apps/kustomization.yaml` uses `kustomize.config.k8s.io/v1beta1` — plain Kustomize
- `flux/flux-system/apps-kustomization.yaml` uses `kustomize.toolkit.fluxcd.io/v1` — Flux CRD
- `prune: false` is mandatory for Phase 1 — enabling prune before Flux owns existing resources would delete live workloads
- `dependsOn` ensures Flux controllers are healthy before attempting to reconcile app HelmReleases

After creating files, validate the plain kustomize config:
```bash
cd flux/apps && kustomize build . 2>/dev/null || kubectl kustomize . 2>/dev/null
```
(This may not work without kustomize installed, but the file structure is simple enough to validate by inspection.)
